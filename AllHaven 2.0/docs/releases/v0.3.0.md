# v0.3.0 — Reasoning Quality Layer

**Date:** 2026-06-09
**Type:** Minor (new feature)

Adds a **Reasoning Quality Layer** so answers are grounded in the user's input,
calculations are checked, irrelevant critique is rejected, and the final answer
honestly reports its confidence — instead of looking active but reasoning weakly.

## Highlights
- A deterministic, model-independent layer that **verifies and scores** every answer.
- A role-based **reasoning council**: Analyst → Critic → Synthesizer, with a quality
  gate and a single retry when the answer scores low.
- A new **Reason** chat mode with Fast / Balanced / Deep depth.

## Backend
- `backend/app/services/reasoning/`:
  - `quality.py` — task detection, fact/assumption extraction, numeric verification
    (year-by-year growth, `EBITDA = revenue × margin`, `X% of Y`), Porter Five Forces
    validation (rejects invented forces like *pengadilan*/court), acquisition-direction
    check, and relevance/grounding/calculation/hallucination scoring → honest confidence.
  - `prompts.py` — Analyst/Critic/Synthesizer + system-core templates with business guardrails.
  - `modes.py` — Fast/Balanced/Deep → pipeline depth + sampling temperature.
- `ai_reasoning_service.reasoning_chat` + `POST /ai/chat/reason`: the Synthesizer
  **rejects irrelevant/invented critique**; a low-quality final answer is **retried once**
  with stricter grounding, else returned honestly with its limits.
- Generation params (`temperature`/`top_p`/penalties) now thread through every adapter.
- Quality scores persist in metadata for debugging only; hidden chain-of-thought is
  never stored or shown — only a concise, deterministic reasoning summary.

### Fixed
- Ollama adapter: missing `network_error_message` import (`NameError` on network failure).

## Frontend
- A **Reason** mode with a Fast/Balanced/Deep selector, a "Show reasoning summary"
  toggle, and a **Debug** toggle. Normal view shows only the final answer plus a
  low-confidence/assumptions warning; Debug reveals the Analyst/Critic/Synthesizer
  turns and numeric quality scores.

## Tests
- `tests/test_reasoning.py` (deterministic guardrails: growth, EBITDA, Porter,
  acquisition, relevance, assumptions, retry trigger) and `tests/test_ai_reasoning.py`
  (council orchestration). Backend suite: **100 passing**. Frontend: typecheck +
  production build pass.

## Known limitations
- Free-form reasoning quality still depends on the underlying model; the deterministic
  layer catches the specific failure classes implemented, not arbitrary domain errors.
- The quality gate is wired into the **Reason** council; Parallel/Debate/single-chat
  paths are unchanged.
