# v0.1.0 — Initial AllHaven Command Center

**Date:** 2026-06-09
**Type:** Initial release (baseline)

The first versioned baseline: the full AllHaven Command Center application — a
self-hostable AI command center (FastAPI + PostgreSQL backend, Next.js +
TypeScript + Tailwind frontend). Not an operating system; an app.

## Highlights
- Email/password auth with JWT and per-user **workspaces**; audit logging.
- Responsive dashboard shell (desktop sidebar + mobile drawer) and a ⌘K command palette.

## Backend
- FastAPI app with Alembic migrations (`0001`→`0005`) and a pytest suite.
- Modules: **Tasks** (with checklists), **Notes**, **Finance**, **Drive** (upload with
  path-traversal protection), **Calendar**, **Weather**, **Automations** (draft-only,
  never auto-executed).
- **AI providers (9):** Ollama (local), OpenAI, Anthropic, Gemini, Grok, Blackbox,
  and OpenRouter ×3 — each independently configurable.
- **Parallel multi-agent chat:** fan one message out to up to 3 agents concurrently,
  with isolated failures and honest per-agent status.
- **Honest verification:** a provider is "online" only after a real authenticated
  check; secrets are encrypted at rest and never returned raw.
- Settings persist to the DB and mirror allowed keys to a local `.env` (strict
  allowlist, atomic write, backups).

## Frontend
- Dashboard pages for every module, AI chat with a conversation sidebar + groups,
  and a Settings screen for providers/integrations/privacy.

## Deploy
- Dockerfiles, dev and prod `docker-compose` (prod with Caddy HTTPS), and the
  `allhaven.sh` one-command helper.

## Notes
- Provider replies require a valid key (or a running local Ollama); the app never
  fabricates AI output.
