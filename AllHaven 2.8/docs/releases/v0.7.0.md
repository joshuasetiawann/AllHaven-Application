# v0.7.0 — Public-launch auth hardening

**Date:** 2026-06-10
**Type:** Minor (security)
**Archive folder:** `AllHaven 1.9` on the `master` branch.
**Audit:** [`LAUNCH_SECURITY_REPORT.md`](../../LAUNCH_SECURITY_REPORT.md)

## What changed

### Cookie sessions (browser) — no more tokens in localStorage
- Login/register set an **HttpOnly** `allhaven_session` cookie (`SameSite=Lax`,
  `Secure` outside local dev, 7-day TTL via `SESSION_TTL_DAYS`).
- The session secret is stored **hashed (SHA-256)** in the new `user_sessions`
  table (migration `0006`) — a DB leak doesn't leak usable sessions.
- `POST /auth/refresh` **rotates** the secret + CSRF token and extends expiry;
  the pre-rotation cookie stops working immediately.
- `POST /auth/logout` **revokes server-side** and clears both cookies — a copied
  cookie dies with the session.
- The frontend stores **no token anywhere**; it caches only the non-sensitive
  user profile and verifies auth with `GET /auth/me` on shell mount (protected
  routes survive refresh without exposing anything to JS). A legacy-key scrub
  removes `allhaven_token` left by older versions.
- **Bearer JWT still works** for API clients, tools, and tests.

### CSRF (double-submit)
- Each session has its own CSRF token in a **readable** `allhaven_csrf` cookie.
- Cookie-authenticated **state-changing** requests must echo it in
  `X-CSRF-Token`; mismatch → `403 CSRF_FAILED`. Enforced centrally in
  `get_current_principal`. Bearer requests are exempt (cross-site pages can't
  set custom headers without passing CORS).
- `/auth/refresh` requires CSRF; `/auth/logout` deliberately doesn't (forced
  logout is an annoyance, a reliable "get me out" matters more).

### Rate limiting + SECRET_KEY guard
- Per-IP sliding-window cap on `/auth/*` POSTs: `AUTH_RATE_LIMIT_PER_MINUTE`
  (0 = off locally; prod example `10`; 429 with the standard error envelope).
  In-memory/single-instance — multi-replica deployments should ALSO limit at
  the gateway.
- Production/staging startup **fails** if `SECRET_KEY` is the dev default or
  shorter than 32 characters.

## Cookie settings (reference)
| Cookie | HttpOnly | SameSite | Secure | Purpose |
|--------|----------|----------|--------|---------|
| `allhaven_session` | ✅ | Lax | prod ✅ / local dev ❌ (http) | the session secret |
| `allhaven_csrf` | ❌ (by design) | Lax | same | double-submit CSRF value |

## Migration
```bash
cd backend && source .venv/bin/activate && alembic upgrade head   # adds user_sessions
```
Users stay compatible: old localStorage tokens are ignored and scrubbed; users
simply sign in once and get a cookie session.
