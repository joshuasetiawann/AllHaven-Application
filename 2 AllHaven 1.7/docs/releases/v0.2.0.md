# v0.2.0 — Multi-agent Debate

**Date:** 2026-06-09
**Type:** Minor (new feature)

Adds an opt-in **Debate** mode alongside the existing parallel fan-out, so multiple
agents can challenge and improve each other instead of only answering side by side.

## Highlights
- A **Parallel ⇄ Debate** toggle in the AI chat header (Parallel stays the default).
- With 2–3 agents: each answers independently (round 1), then critiques and refines
  against the others' answers across rebuttal rounds, and the first selected agent
  **synthesizes a single best final answer**.

## Backend
- `ai_debate_service.debate_chat` + `POST /ai/chat/debate`.
- Concurrent per-round fan-out (network calls in worker threads, all DB access on the
  request thread); failures stay isolated.
- Honest handling: needs ≥2 runnable agents to truly debate; 1 agent returns its own
  answer; 0 is an honest error. Non-runnable agents are shown with their real status.
- Rounds and the synthesis persist via existing tables' `meta` columns — **no DB migration**.

## Frontend
- Rounds selector (2–3) and a responsive debate transcript: per-round agent cards
  (`grid-cols-1 sm:2 xl:3`) with a highlighted synthesized final answer; rebuilt on
  reload from message metadata.

## Tests
- `tests/test_ai_debate.py`: rounds, isolated failure, synthesis, single/no-agent,
  and that raw secrets are never returned.
