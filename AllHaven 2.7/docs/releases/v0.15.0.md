# v0.15.0 — Premium UI polish, persistent model selection & per-section chat memory

**Date:** 2026-06-12 · **Type:** Minor · **Archive:** `AllHaven 2.7` on `master`.

A focused **frontend / UX** release. No backend, API, or database changes — every
improvement lives in `frontend/`. The goal: make the Command Center feel calmer,
faster, and more premium, and stop making you re-pick your AI model on every screen.

## What changed

### Persistent model selection — never re-pick again
- Your **selected agents/model**, **chat mode** (Parallel / Debate / Reasoning),
  **Thinking depth**, and **debate rounds** are now remembered across navigation and a
  full browser refresh (stored locally under `allhaven:ai-selection`).
- On load the saved selection is **reconciled against the providers actually available**:
  - models that disappeared (e.g. a provider was disabled) are dropped with a clear
    "saved model unavailable — switched to a default" notice;
  - when nothing is configured you get **"Configure an AI provider first"** with a link
    to Settings.
- The **workspace default mode** only applies on a fresh device (no saved choice yet),
  so your own last choice always wins afterward.

### Per-section chat memory
- Each module section — **General, Tasks, Notes, Finance, Calendar, Files, Automations,
  Weather, Settings, System Control** — and **each chat project/group** carries its own
  **local, editable memory**: a `title`, a `summary`, and important-context bullets.
- Pick the active section from the chat header, then **view / edit / clear** its memory.
  Switching sections keeps each memory isolated; it survives refresh.
- The active section's memory is **injected into the prompt once per thread**, so answers
  stay relevant without repeating context every turn.
- **Privacy:** every value is passed through a **secret-redactor** before saving
  (API-key / token / bearer / JWT patterns become `•••redacted•••`). Clear one section or
  all local memory at any time.
- Backed by a new **local storage abstraction** (`lib/storage.ts`) — namespaced,
  versioned, SSR-safe, and ready to move to IndexedDB or a server table behind one
  interface.

### Smooth, restrained animations
- Page/route transitions, dropdown & popover entrances, chat **message-in**, and the
  pending-actions panel now animate in at **120–250ms** with a soft `easeOut` curve.
- A global **`prefers-reduced-motion`** guard collapses all animation/transition for
  users who ask their OS to reduce motion.

### Polish & fixes
- **Finance** KPI cards: clearer hierarchy with tinted icon chips, **tabular figures**,
  and a hero **Balance** card that turns red when negative.
- **Settings**: tab switches fade in.
- **Fixed:** the **"Checking your session…" flash** that appeared on every dashboard
  navigation — the session is now confirmed once per page-load.

## Upgrade notes

Nothing to migrate. Pull, rebuild the frontend (`npm install && npm run build` in
`frontend/`, or just relaunch Haven), and the new local preferences/memory start empty.
All data stays on your device; no secrets are ever written to local memory.
