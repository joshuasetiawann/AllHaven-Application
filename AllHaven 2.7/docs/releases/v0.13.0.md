## The flow

1. User runs a terminal command: `./START_HAVEN_LINUX.sh` (or the Windows/macOS
   launcher), or `./install.sh`, or `npm run setup`.
2. The bootstrapper checks the minimal dependency (**Python 3**).
3. It starts the local setup server (`installer/haven_setup.py`, localhost, token-gated).
4. It **auto-opens the browser** to the GUI installer (`http://127.0.0.1:7000`).

After setup, the user opens Haven from the **desktop shortcut** with no terminal. If a
service is down, the shortcut runs the launcher that checks and safely starts services
first, then opens the app.

## What changed

- **Launchers branch on setup state** (`START_HAVEN_*`):

- **New entry points:** `install.sh` (Linux/macOS bootstrapper â†’ wizard) and a root
  `package.json` providing `npm run setup` / `npm run start` via a cross-platform node

  through the logged path so the image pull is visible.
