# AllHaven 3.4.0

Release date: 2026-06-13

## Summary

AllHaven 3.4 improves everyday use before launch: AI Chat can take voice
dictation and document attachments, AI Knowledge can extract PDF/DOC/DOCX text,
Routine has a calmer agenda layout, and Supabase sync now mirrors the main
workspace data after local writes.

## Added

- Voice note input in AI Chat using browser speech recognition when supported.
- PDF, DOC, DOCX, text, and code upload from AI Chat into AI Knowledge.
- Built-in DOCX parser (hardened with `defusedxml` against XXE / entity
  expansion), PDF fallback parser with optional `pypdf` support, and
  best-effort legacy DOC extraction.
- Workspace-wide local-first Supabase mirroring for:
  - tasks and checklist items
  - notes
  - finance categories and transactions
  - routine events
  - weather locations
  - drive file metadata
  - automations
  - provider and integration configuration rows
  - chat groups, sessions, messages, multi-agent runs, and agent responses
  - AI memories, suggestions, summaries, tool proposals, and tool calls
  - AI Knowledge documents and chunks
  - audit logs

## Changed

- Routine is redesigned as an agenda/timeline with lighter rows instead of a
  box-heavy board.
- Add Routine is simplified and no longer asks for color or logo/icon.
- AI Routine tool schemas now focus on title, time, repeat, days, location, and
  notes.
- AI Knowledge UI copy now explains PDF/DOC/DOCX support and metadata fallback.

## Safety and sync model

- The local database remains the source of truth.
- Supabase sync runs after successful local writes and never blocks the user.
- Missing Supabase tables are skipped per table so the rest of the mirror can
  continue.
- Password hashes, browser session token hashes, and raw Drive file bytes are
  not mirrored. Provider/integration secrets are mirrored only as already
  encrypted database blobs.

## Verification

- Added tests for PDF, DOCX, and legacy DOC knowledge ingestion, including a
  regression test that a malicious DOCX (DTD/entity expansion) is refused and
  stored metadata-only rather than parsed.
- Added AI Tool Registry coverage tests for the core modules.
- Updated Supabase sync tests for the broader workspace mirror.
