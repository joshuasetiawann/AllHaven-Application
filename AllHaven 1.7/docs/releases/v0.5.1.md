# v0.5.1 — Honest "model can't read images" status

**Date:** 2026-06-10
**Type:** Patch (fix)
**Archive folder:** `AllHaven 1.7` on the `master` branch.

## The problem
Vision support depends on the **model**, not just the provider. A provider can be
vision-capable in general (Ollama, OpenRouter, …) while the **selected model** is
text-only. When that happened, the agent card showed the raw provider error:

- Ollama (`llama3.1`): `HTTP 400 — "Multimodal data provided, but model does not support multimodal requests."`
- OpenRouter (`meta-llama/llama-3.1-8b-instruct`): `HTTP 404 — "No endpoints found that support image input"`

## The fix
When an image is attached and a provider returns a "can't do images" error, it is
now reclassified to the honest **`unsupported`** status with actionable guidance —
the same clean "No image support" card used for non-vision providers — instead of
leaking raw API JSON. Applies to **Parallel, Debate, and Reasoning**.

## How to actually get image answers
Set a **vision model** in **Settings → AI Providers**:
- **Ollama:** `ollama pull llava` (or `llama3.2-vision`) and set it as the default model.
- **OpenRouter:** use a vision model, e.g. `openai/gpt-4o-mini`, `google/gemini-2.0-flash-001`, `anthropic/claude-3.5-sonnet`.
- **GPT / Claude / Gemini agents:** already vision-capable with their default models.

Vision-capable providers are marked with an **eye** icon in the agent picker.
