# v3.2.0 — AllHaven 3.2 Repository Hygiene and Render Skeletons

**Date:** 2026-06-13 · **Type:** Product polish release · **Product version:** `AllHaven 3.2`.

## Summary

AllHaven 3.2 focuses on keeping the repository cleaner and making page renders feel more polished.
The latest app now has route-level skeleton loaders, version metadata is synchronized, and the
archive branch is prepared to distinguish the early CoreOS era from the AllHaven releases.

## Added

- App-wide **route render skeletons** for Next.js App Router navigation and loading states.
- A dashboard-specific skeleton shell with sidebar, topbar, header, stat cards, chart blocks, and side panels.
- A global `shimmer` keyframe so skeleton elements animate consistently instead of staying static.

## Changed

- `Skeleton` now accepts normal HTML div attributes, making it reusable for responsive loader shapes.
- Version metadata is bumped to `3.2.0` across the root package, frontend package and lockfile,
  backend package metadata, `/VERSION`, and the sidebar app version.

## Repository Hygiene

- Local environment files were removed from the working tree:
  `.env`, `.env.bak.*`, `backend/.env`, and `frontend/.env.local`.
- Template files remain intentionally present for new clones:
  `.env.example`, `.env.prod.example`, and `frontend/.env.local.example`.
- `.gitignore` already protects local env files, dependency folders, build outputs, and runtime data.

## Archive Notes

- The `master` archive for this release records the early `0.1` through `1.3` line as **CoreOS**.
- CoreOS semantic mapping is clarified:
  - `CoreOS 1.2` maps to `v0.1.0`.
  - `CoreOS 1.3` maps to `v0.2.0`.
  - `AllHaven 1.4` starts the AllHaven product line at `v0.3.0`.
