# v0.8.0 — Live n8n workflows in Automations

**Date:** 2026-06-10
**Type:** Minor (feature)
**Archive folder:** `AllHaven 2.0` on the `master` branch.

## What changed
The **Automations** page now talks to your **connected n8n** instead of only
holding local drafts.

- **List** your real workflows: `GET /api/v1/n8n/workflows` calls n8n's public API
  (`GET /api/v1/workflows`) with the workspace's Base URL + API key.
- **Activate / deactivate** from AllHaven: `POST /api/v1/n8n/workflows/{id}/active`
  → n8n `activate`/`deactivate`.
- **Open in n8n**: a direct link to `{base}/workflow/{id}`.
- Local draft definitions are kept, relabeled **"Local drafts — not executed"**.

## How to use it
1. **Settings → Connected Tools → n8n:** set the **Base URL** and your **API key**
   (n8n → Settings → API; the **Public API** must be enabled).
2. Open **Automations** — your workflows appear with Active/Inactive toggles and
   an *Open in n8n* link. Use **Refresh** after changing things in n8n.

## Honest states (no faking)
| Status | Meaning |
|--------|---------|
| `online` | workflows loaded |
| `not_configured` | n8n Base URL not set |
| `no_api_key` | Base URL set but no API key (enable n8n Public API + add the key) |
| `unauthorized` | n8n rejected the API key |
| `unavailable` | n8n unreachable |
| `error` | n8n returned an error (e.g. Public API off) |

There is **no generic "run"**: the n8n public API can't execute an arbitrary
workflow (only webhook-triggered flows can be fired by URL), so we don't offer a
fake Run button.

## Security
- The n8n **API key stays server-side** (decrypted only to call n8n; never in any
  JSON response — covered by a test).
- Workspace-scoped: each workspace uses its own n8n config.
