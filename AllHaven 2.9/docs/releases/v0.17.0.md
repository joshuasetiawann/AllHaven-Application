# v0.17.0 — AI Workspace, Knowledge, Finance Reports, and Direct Memory

**Date:** 2026-06-13 · **Type:** Minor · **Archive:** `AllHaven 2.9` on `master`.

This release turns AllHaven's AI layer into a more complete workspace assistant:
section-aware context, AI Knowledge documents, audited tool calls, better memory,
finance reports, local time/date answers, and a cleaner approval experience.

## What changed

### AI Workspace context
- Added a shared context builder for chat requests.
- Context now includes the active section, thinking mode, tool priorities, relevant
  memories, conversation summaries/snippets, and AI Knowledge retrieval when useful.
- Single chat, one-agent Parallel, Debate, and Reasoning now receive the same
  workspace-aware context instead of each mode building its own partial prompt.

### AI Knowledge
- Added a first-class AI Knowledge module.
- Backend support includes document metadata, chunk storage, indexing, search, and
  retrieval tools.
- Frontend support includes an AI Knowledge page and API wiring.
- Text extraction currently supports `.txt`, `.md`, and `.csv`; unsupported formats
  are stored honestly with indexing status instead of pretending to be searchable.

### Tool Registry and audit logging
- Added `ai_tool_calls` for dedicated tool-call history with redacted arguments and
  result previews.
- Expanded the Tool Registry to 72 allowlisted tools.
- New/expanded tools cover conversation history, task/note draft aliases, finance
  summaries, file metadata, AI Knowledge search/retrieval, system status/logs, and
  local date/time.
- Unknown or disabled tools return honest errors and are audited.

### Finance reports
- Added report-scoped finance queries and `/finance/report`.
- The Finance page now supports monthly and weekly report modes, period navigation,
  trend bars, and report-specific income/expense/balance totals.
- Older transactions outside the current report are shown as archived/out-of-period
  records, so a 2023 record is no longer confused with a 2026 report.
- Old records can be moved into the active month/week from the UI.
- AI-created transactions default to the app's current date when the user does not
  specify a date.

### Memory
- Low-risk memory create/update/disable actions now save directly so preferences can
  be remembered without noisy approvals.
- Higher-risk or destructive memory actions still require human approval.
- Memory extraction now recognizes direct-response preferences and explicit AI usage
  needs such as coding help and schedule management.
- Memory ranking prioritizes profile, preferences, writing style, work context, and
  relevant active memories.
- The memory page shows richer metadata and supports cleaner editing.

### Chat and assistant behavior
- The assistant now answers local time/date questions through deterministic backend
  logic, even when no AI provider is configured.
- System prompts are tuned for direct Indonesian responses, casual chat when invited,
  jokes when appropriate, serious work mode, senior coding assistance, and schedule
  or calendar help.
- Pending action notifications are cleaner: approval/rejection feedback appears as a
  short toast-style notice, while the pending panel stays focused on unresolved work.

### Drive
- Added configurable upload limits through `DRIVE_MAX_UPLOAD_MB`.
- Backend and frontend both read the configured limit; the default is 250 MB.
