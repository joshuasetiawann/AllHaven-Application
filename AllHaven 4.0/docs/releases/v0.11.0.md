Addresses three issues reported on the v0.10.0 launch flow:

1. Generating a new `.env` didn't also update `backend/.env`.
2. The browser wizard's **"Checking Docker"** step felt stuck (a first-run image
   pull happens with no visible progress), even though running manually works.
3. A request to be able to **install from the terminal** instead of the web.

## 1. Terminal installer (`installer/haven_cli.py`) â€” now the default

The `START_HAVEN_*` launchers now run a terminal-first installer by default. It
prints **live progress** for the slow steps right in your terminal â€” which is
exactly what made the manual flow feel responsive:

- checks tools (Python / Node / npm / Docker) with a bounded, fast Docker probe;
- configures `.env` (creating it with fresh secrets if missing, optional port
  prompts on first run) and mirrors it to `backend/.env`;
- **`docker compose up -d postgres` with streamed output** (you see the image
  pull), then waits for the DB;
- backend venv + `pip install` and `alembic upgrade head` (streamed);
- frontend `npm install` (streamed);
- then hands off to the proven launcher to start everything and open the browser.

It is **idempotent** â€” re-running skips finished steps and just starts the app.
The browser wizard is still available: set `HAVEN_SETUP_WEB=1` before launching.

## 2. `backend/.env` follows the root `.env`

`ensure_env_files()` now **mirrors the repo-root `.env` to `backend/.env`** and
rewrites it whenever it differs. This runs in the terminal installer, the web
wizard's *Write config* step, and on every service start â€” so a change you make
(secrets, ports) propagates to the backend with no stale copy. (`frontend/.env.local`
is still created from its example.)

## 3. Faster, non-stalling Docker check

`docker_running()` now uses a lighter `docker version --format '{{.Server.Version}}'`
query with an **8s** timeout (was `docker info`, 12s). Combined with the terminal
installer streaming `docker compose` output, the "is Docker up + pull the image"
phase shows real progress instead of a frozen spinner.

## How to verify
1. From a terminal (or by double-clicking the launcher): `./START_HAVEN_LINUX.sh`
   (or the macOS/Windows equivalent). You should see step-by-step progress,
   including the Docker image pull, `pip`, and `npm`.
2. Confirm `backend/.env` exists and matches `.env` after setup.
3. The app opens and the login page reaches the API.
4. To try the old browser wizard: `HAVEN_SETUP_WEB=1 ./START_HAVEN_LINUX.sh`.

## Known limitations
- First-run install needs network access for the Docker image, `pip`, and `npm`.
- Changing the **backend port** later still doesn't rewrite
  `frontend/.env.local`'s `NEXT_PUBLIC_API_BASE_URL` (default 8000 is handled).
