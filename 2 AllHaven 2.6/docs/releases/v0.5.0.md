# v0.5.0 — Calculator, Clock, Thinking Mode & vision routing

**Date:** 2026-06-09
**Type:** Minor (features)
**Archive folder:** `AllHaven 1.6` on the `master` branch.

## Backend dependency fix
`ModuleNotFoundError: No module named 'fastapi'` happens when the backend runs
without its virtualenv activated / dependencies installed. The dependency set is
correct (`fastapi` is pinned in `backend/requirements.txt`). Exact run command:

```bash
cd backend
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
alembic upgrade head               # first run only (needs PostgreSQL)
uvicorn app.main:app --reload --port 8000
```

Verify: `python -c "from app.main import app"` imports cleanly (86 routes).

## New modules
- **Calculator** (`/dashboard/calculator`): add, subtract, multiply, divide,
  percent, ±, decimal, clear, backspace; full keyboard support; responsive.
- **Clock** (`/dashboard/clock`): live time/date/timezone, **stopwatch** (laps),
  **timer** (countdown + alert), and an **alarm** foundation (saved in the
  browser, rings while the app is open).
- Both are linked in the sidebar and match the premium dark theme.

## Chat: simplified modes + Thinking Mode
- Chat Mode is now exactly **Parallel · Debate · Reasoning**.
- **Thinking Mode** lives near the input and applies to every chat mode:

  | Mode | temperature | top_p | reasoning depth |
  |------|------------|-------|-----------------|
  | Fast | 0.35 | 0.85 | 1 pass |
  | Balance (default) | 0.25 | 0.80 | analyst + synthesizer |
  | Thinking | 0.15 | 0.75 | + critic |
  | Deep | 0.10 | 0.70 | + critic (max depth) |

## Image input + vision routing
- Provider capability metadata: `supports_text` / `supports_image` / `supports_tools`,
  exposed in `GET /ai/providers`. Vision-capable models show an **eye** icon.
- Images route **only** to vision-capable providers. A non-vision provider returns
  an honest **`unsupported`** status ("this model can't read images…") instead of
  guessing. The composer warns before sending and confirms when vision-ready.
- Drag-and-drop image upload; preview + remove; up to 4 images; client-side downscale.
- Per provider: OpenAI/OpenRouter/Grok image_url, Anthropic image blocks, Gemini
  inline_data, Ollama base64 images; Blackbox is marked non-vision (honest).

## Tests & verification
- Backend: **116 passing** — new `test_upgrade.py` covers Thinking Mode params,
  reasoning-depth mapping, provider capabilities, image→non-vision `unsupported`,
  thinking-mode generation settings, and the calculator evaluator.
- Frontend: typecheck + production build pass (Calculator, Clock, chat all compile).

## Notes / limitations
- Calculator computes client-side (instant); a safe server-side evaluator
  (`calc_service`, no `eval`) backs the tests.
- Alarms are a foundation: browser-local and ring only while the page is open.
