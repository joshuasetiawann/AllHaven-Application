# AllHaven 3.8.0

**Mobile full release, unified accounts, performance, and Weather removal.** 2026-06-19.

A hardening + product release on top of 3.7's Supabase foundation: one account works
identically on desktop and mobile, the mobile app ships the full feature set again,
several reliability and performance bugs are fixed, and the Weather feature is removed.

## Highlights

### One account across desktop + mobile
Desktop (local password auth) and Supabase Auth used to be two separate credential
stores, which could drift and required a manual "Connect to Supabase" step. Now a
successful **desktop login** best‑effort syncs the Supabase Auth password to the
desktop password and links the profile (`profiles.supabase_user_id`), on a daemon
thread so login stays fast. Registration already provisions the same password, so new
accounts are unified from day one. Net result: the **same email + password** signs you
in on both platforms — no extra step.

### Mobile is a full release
The previous build hid the REST‑backed features on mobile; they're back. AI Chat,
Settings, Drive, AI Knowledge/Memory, and Approvals are all available again. (These
talk to the backend over Tailscale until the planned local‑first mobile store lands;
when unreachable they now fail fast rather than hang.)

### Performance & reliability
- REST request timeout on mobile cut from **20s → 6s**; the dashboard renders Supabase
  data immediately and loads integrations in the background; the Topbar stops
  re‑polling an unreachable backend.
- `loadMe` uses `maybeSingle` with clear `PROFILE_NOT_INITIALIZED` / `WORKSPACE_NOT_INITIALIZED`
  errors; PostgREST `PGRST116` ("can't update by id") maps to an actionable message;
  workspace‑scoped writes are guarded with a clear "sign in again".
- **Desktop CSS fix:** a `predev` clean stops the mobile static export from leaving a
  mismatched manifest in `next dev`'s `.next`, which had served unstyled pages.
- **Migration fix:** revision `0015`'s id was shortened to fit Alembic's
  `alembic_version.version_num VARCHAR(32)`, so `alembic upgrade head` applies cleanly
  on both the local Postgres and Supabase.
- **AI approval safety:** `create_transaction` proposals normalize empty
  `category_id`/`transaction_date`; the approval card shows readable, editable fields;
  `create_user`/"Connect to Supabase" is idempotent.

### Removed
- **Weather** — removed from the UI, API endpoints, AI tool, and integration settings.
  The `weather_locations` table/model is retained (dormant) so existing data and the
  sync watermarks are untouched; removing it would require a migration.

## Upgrade notes
- After deploying, **log in once on desktop** to sync your Supabase password, then sign
  in on mobile with the same credentials.
- Rebuild the Android APK to pick up the full‑feature mobile UI and the timeout/login
  fixes.
