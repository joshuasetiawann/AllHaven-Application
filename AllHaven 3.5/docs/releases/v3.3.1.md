# v3.3.1 — AllHaven 3.3.1 Local Routine UX Polish

**Date:** 2026-06-13 · **Type:** Patch UX release · **Product version:** `AllHaven 3.3.1`.

## Summary

AllHaven 3.3.1 polishes Routine into a clearer local-first planning surface.
The page no longer feels like a Google Calendar connector, and the routine form
is lighter for day-to-day use.

## Added

- Habit-style Routine builder fields:
  - icon,
  - color,
  - repeat rule,
  - repeat days,
  - Pagi, Siang, and Malam time slots.
- Migration `0009_routine_preferences` stores routine preferences locally on
  existing workspace schedule records.

## Changed

- Removed the routine goal field to keep creation focused and faster.
- Routine board now keeps Pagi, Siang, and Malam lanes visible even when empty,
  giving users clear add targets.
- Routine copy now explicitly says schedules are stored in the AllHaven backend
  database and do not require Google Calendar.
- AI routine tools now understand the new local routine metadata fields.

## Fixed

- Routine loading now falls back to the compatible local `/calendar/events`
  endpoint if the frontend is updated before the backend has restarted with
  `/routines/events`.
