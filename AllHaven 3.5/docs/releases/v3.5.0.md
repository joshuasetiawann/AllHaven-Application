# AllHaven 3.5.0

Release date: 2026-06-14

## Summary

AllHaven 3.5 adds AI assistance to Routine. Describe your day and the
workspace's configured AI provider drafts a short, realistic set of routine
items for the chosen time-of-day window. You review and edit every draft, then
save them together in one atomic batch. Generation is honest by design — it
never fabricates routines, never saves on its own, and tells you plainly when a
provider is missing or turned off.

## Added

- **AI routine generation** (`POST /routines/generate`). A prompt plus a
  Morning / Afternoon / Evening window asks the configured provider for at most
  eight routine ideas. Your open tasks and existing routines for that day are
  passed as optional context (best-effort, never fabricated). The endpoint is
  generate-only: it returns drafts and never writes to the database.
- **Atomic batch save** (`POST /routines/events/batch`). Reviewed drafts are
  saved together. Every item is built and validated before anything is inserted,
  so an invalid item rolls back the whole batch — you never end up with a
  partially-applied save. Capped at 50 items per request.
- **Routine sync status card** (`GET /routines/sync-status`). The Routine
  summary shows whether the workspace Supabase mirror is `active` or running
  `local_first`. The endpoint degrades gracefully and never blocks the page.

## Changed

- The routines dashboard page is refactored from one large file into focused
  components: `DateStrip`, `PeriodSection`, `SummaryCards`, `RoutineToolbar`,
  `RoutineEmptyState`, `RoutineSkeleton`, `GenerateModal`, `RoutineFormModal`,
  and a shared helpers module.

## Honesty and safety model

- **Generate-only.** `routine_ai_service.generate_drafts` never writes to the
  database under any branch; drafts are saved only when the user explicitly
  confirms the batch.
- **No fabrication.** With no provider configured the caller gets a clear
  `not_configured` state; a configured-but-disabled provider keeps its specific
  "enable it in Settings" message (`blocked`); unparseable model output returns
  an honest `error` instead of a silent empty success.
- **Server-side validation.** Draft times are snapped into the requested
  Morning/Afternoon/Evening window, the count is capped at 8, and title, field
  lengths, repeat rule, and weekday values are validated before drafts reach the
  review modal.
- **Workspace-scoped context.** Task and routine context is read only through
  the principal-scoped services, so generation cannot pull another workspace's
  data.
