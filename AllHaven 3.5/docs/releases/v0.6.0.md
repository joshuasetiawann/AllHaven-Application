## Backend dependency / run
Backend imports and starts cleanly (86 routes). The `ModuleNotFoundError: fastapi`
only occurs when the virtualenv isn't active. Exact command:

```bash
cd backend
python -m venv .venv && source .venv/bin/activate     # Windows: .venv\Scripts\activate
pip install -r requirements.txt
alembic upgrade head            # first run (needs PostgreSQL)
uvicorn app.main:app --reload --port 8000
```

## Security hardening
- **HTTP security headers** everywhere — backend middleware (`nosniff`, `X-Frame-Options: DENY`, `Referrer-Policy`, `Permissions-Policy`) and frontend `next.config.js` (same set + production **CSP** with `frame-ancestors 'none'`, `object-src 'none'`, no `unsafe-eval`). CSP is production-only so dev HMR keeps working; no CSP on the backend so `/docs` (Swagger) still loads.
- **Safe file downloads** — Drive forces `Content-Disposition: attachment` and serves active content types (HTML, SVG, JS, XML) as `application/octet-stream`, closing an inline-XSS vector. Path traversal, size cap (25 MB), UUID names, and workspace scoping were already in place.
- **Dependency patches** — Next `14.2.18 → 14.2.35` (fixes the **critical** middleware authorization bypass) and `postcss → 8.5.15` tree-wide via an `overrides` (fixes the moderate stringify-XSS). One residual Next "high" (DoS/cache/SSRF in features this app doesn't use) is documented with exposure analysis — staying on 14.2.x avoids a breaking Next 16 / React 19 migration.
