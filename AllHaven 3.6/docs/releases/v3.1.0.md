# v3.1.0 — AllHaven 3.1 Expanded AI Agents and Settings UX

**Date:** 2026-06-13 · **Type:** Major product release · **Product version:** `AllHaven 3.1`.

This release expands the AI workspace from a 7-agent setup to a 10-agent setup,
adds Cursor-compatible, DeepSeek, and Qwen providers, improves Debate output, and
cleans up Settings so model setup is easier to scan.

## Highlights

- Multi-agent Parallel, Debate, and Reasoning now allow up to **10 agents**.
- Added **Cursor AI** as a Cursor/OpenAI-compatible coding provider with `Cursor 1`
  and `Cursor 2` model slots. It requires an explicit compatible base URL to avoid
  false online status.
- Added first-class **DeepSeek** and **Qwen** provider support through
  OpenAI-compatible adapters.
- AI Chat's model picker is searchable and grouped into **Direct model agents** and
  **OpenRouter agents**.
- Settings → AI Providers now has compact health stats, clearer sections, slot
  previews on provider cards, and a wider configure modal.
- Debate prompts are more direct, carry context through rebuttal rounds, and produce
  cleaner final answers with fewer repeated panel details.

## Honest Status

AllHaven still never fakes provider health. Saving a key marks the provider as
`configured`; `online` is set only after a real **Test Connection** succeeds.
Cursor AI is intentionally modeled as a compatible gateway provider because Cursor's
public API is agent/workflow-oriented rather than a normal chat-completions endpoint.

## Configuration Keys

New local `.env` keys:

```dotenv
CURSOR_API_KEY=
CURSOR_DEFAULT_MODEL=
CURSOR_BASE_URL=
DEEPSEEK_API_KEY=
DEEPSEEK_DEFAULT_MODEL=
QWEN_API_KEY=
QWEN_DEFAULT_MODEL=
```
